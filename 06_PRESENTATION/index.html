Identificacao do Problema

Por que este projeto existe?

Pequenas e medias empresas brasileiras enfrentam um dilema: precisam proteger dados de clientes conforme a LGPD, mas nao tem equipe de seguranca dedicada nem orcamento para ferramentas caras.

Bucket S3 publico

Dados de clientes expostos na internet. Multa de ate 2% do faturamento (LGPD, Art. 52).

Usuarios sem MFA

Une credencial vazada compromete toda a conta AWS. Sem rastreabilidade de acesso.

Logs desativados

Sem CloudTrail, e impossivel saber quem fez o que. LGPD Art. 37 exige rastreabilidade.

Sem criptografia

Dados em repouso sem criptografia violam diretamente o Art. 46 da LGPD.

Solucao Tecnica

"Glass Box Architecture"

Cada acesso a dados sensiveis e registrado, criptografado e monitorado. Nao e uma caixa preta — e uma caixa de vidro: voce ve tudo que acontece dentro.

Script 1: Auditoria de Criptografia S3 (Art. 46)

Verifica se cada arquivo de log esta criptografado com aws:kms antes de aceitar como evidencia valida.

04_CODE/validate_audit_logs.py
import boto3
import hashlib
from datetime import datetime

# LGPD Art. 46: soberania de dados na regiao brasileira
REGION = "sa-east-1"

def validate_log_encryption(bucket_name, key):
    """Verifica se o arquivo esta criptografado em repouso (Art. 46)."""
    s3 = boto3.client('s3', region_name=REGION)
    response = s3.head_object(Bucket=bucket_name, Key=key)
    encryption = response.get('ServerSideEncryption')
    return encryption in ['AES256', 'aws:kms'], encryption

def audit_trail_compliance():
    compliance_score = 0
    sample_logs = [
        "AWSLogs/123456789/CloudTrail/sa-east-1/log-a.json.gz",
        "AWSLogs/123456789/CloudTrail/sa-east-1/log-b.json.gz"
    ]
    for log_file in sample_logs:
        is_encrypted, algo = (True, "aws:kms")  # simulado
        status = "[PASS] COMPLIANT" if is_encrypted else "[FAIL]"
        print(f"Verificando: {log_file} -> {status} ({algo})")
        if is_encrypted:
            compliance_score += 1

    # Salvar evidencia conforme Art. 37 (rastreabilidade)
    with open("compliance_report.txt", "w") as f:
        f.write(f"AUDIT REPORT - {datetime.now()}\n")
        f.write(f"Score: {compliance_score}/{len(sample_logs)}\n")

class IAMSecuritySetup:

    def setup_password_policy(self):
        """Politica de senhas alinhada AWS Security Best Practices."""
        self.iam_client.update_account_password_policy(
            MinimumPasswordLength=14,   # acima do minimo recomendado
            RequireSymbols=True,
            RequireNumbers=True,
            RequireUppercaseCharacters=True,
            ExpirePasswords=True,
            MaxPasswordAge=90,           # rotacao a cada 90 dias

Servicos AWS Utilizados

Cada servico tem um papel especifico

O Holocron Sentinel nao usa a AWS de forma generica. Cada servico foi escolhido para resolver um requisito especifico de seguranca e conformidade com a LGPD.

IAM

AWS Identity and Access Management

Controla quem pode acessar quais recursos. Criamos 3 grupos com permissoes diferentes: Admins, Developers e ReadOnly. Nenhum usuario tem mais acesso do que precisa.

Governanca de Identidade
S3

Amazon Simple Storage Service

Armazena os arquivos de log e evidencias de auditoria. Configurado com criptografia SSE-KMS obrigatoria, bloqueio de acesso publico e versionamento. Cumpre o Art. 46 da LGPD.

Protecao de Dados
CT

AWS CloudTrail

Registra toda e qualquer acao realizada na conta: quem logou, quem criou um usuario, quem apagou um arquivo. Ativado em todas as regioes para garantir rastreabilidade total (Art. 37 LGPD).

Auditoria e Rastreabilidade
CW

Amazon CloudWatch

Monitora metricas e cria alarmes automaticos. Configurado para notificar o time quando alguem tenta fazer login sem MFA ou quando o gasto da conta supera o limite definido.

Monitoramento em Tempo Real
KMS

AWS Key Management Service

Gerencia as chaves de criptografia. As chaves ficam no KMS, nunca nos aplicativos. Isso significa que mesmo que alguem acesse o bucket S3, os dados continuam ilegíveis sem a chave.

Criptografia e Chaves
CFG

AWS Config

Verifica continuamente se as configuracoes estao dentro das regras. Se alguem desativar a criptografia de um bucket S3, o AWS Config detecta e registra a violacao automaticamente.

Script: ai_compliance_analyst.py

Le o relatorio de auditoria e envia ao modelo de linguagem, que retorna um parecer estruturado em portugues. Em producao AWS, a chamada ao Gemini seria substituida por uma chamada ao Amazon Bedrock.

04_CODE/scripts/ai_compliance_analyst.py
import google.generativeai as genai

# Configura o modelo de linguagem
# Em producao AWS: substituir por chamada ao Amazon Bedrock (Claude 3)
genai.configure(api_key=GEMINI_API_KEY)
model = genai.GenerativeModel("gemini-1.5-flash")

def analisar_com_ia(relatorio: str) -> str:
    """Envia relatorio ao LLM e recebe parecer executivo estruturado."""

    prompt = f"""Voce e um Analista Senior de Seguranca em Nuvem, especialista em LGPD e AWS.

Analise o relatorio abaixo e produza um PARECER EXECUTIVO em portugues formal,
adequado para ser entregue a um Diretor de TI.

Regras: (1) Identifique os 3 riscos criticos em ordem de prioridade.
        (2) Para cada risco, descreva o impacto real para o negocio.
        (3) Recomende a acao corretiva com o servico AWS especifico.
        (4) Escreva uma conclusao de 2 linhas para o executivo.

RELATORIO: {relatorio}"""

    response = model.generate_content(prompt)
    return response.text

# Carrega relatorio real e salva parecer gerado pela IA
with open("compliance_report.txt", "r") as f:
    relatorio = f.read()

parecer = analisar_com_ia(relatorio)

with open("parecer_executivo.md", "w") as f:
    f.write(parecer)

Servicos AWS de Inteligencia Artificial para producao

Amazon GuardDuty

Deteccao de ameacas com Machine Learning nativo da AWS. Analisa logs do CloudTrail e VPC sem configuracao de regras manuais. Identifica anomalias como acesso de IPs suspeitos, mineracao de criptomoedas e exfiltracao de dados em tempo real, 24 horas por dia.

Amazon Macie

Varredura automatica dos buckets S3 com IA para localizar dados pessoais: CPFs, documentos e emails. Gera um inventario completo dos dados sensiveis armazenados, essencial para cumprir o Art. 37 da LGPD, que exige o registro das operacoes sobre dados pessoais.

Amazon Bedrock — Claude 3

Plataforma de modelos de linguagem gerenciada pela AWS. Substitui diretamente a API do Gemini usada neste projeto. Os dados processados nao saem da infraestrutura da empresa, garantindo privacidade e conformidade, com custo por token e sem servidor para gerenciar.

AWS Security Hub

Painel centralizado que agrega achados do GuardDuty, Macie e Config. Com integracao ao Amazon Bedrock, o modelo pode explicar um alerta critico em linguagem de negocio e sugerir automaticamente o plano de resposta ao incidente, sem necessidade de analista tecnico dedicado.

Gemini API (demonstracao) vs. Amazon Bedrock (producao)
Google Gemini API — usado agora
Chave gratuita disponivel em aistudio.google.com. Nao exige conta AWS ativa. Permite demonstrar o conceito funcionando. A arquitetura de prompt e resposta e identica ao Bedrock.
Amazon Bedrock — producao AWS
Executado dentro da propria conta AWS. Os dados nao saem da infraestrutura da empresa. Integrado nativamente a IAM, CloudTrail e S3. Cobranca por token consumido, sem servidor para administrar.
